## 一、项目概述

**目标用户**：广西糖业集团红河制糖有限公司压榨车间安全管理人员（iPhone 用户）

**核心场景**：车间日常安全检查，拍摄问题照片 → 语音/文字描述 → AI 润色 → 自动生成固定格式的 Word 整改报告 → 下载或微信分享。

**使用方式**：手机浏览器打开网页即可使用，无需安装 App。

## 三、核心业务规则

### 3.1 固定抬头信息
- **公司名称**：固定"广西糖业集团红河制糖有限公司"，不可更改
- **部门**：固定"压榨车间"，不可更改

### 3.2 日期规则

```
┌──────────────────────────────────────────────────────┐
│  检查日期 ≠ 落款日期                                  │
│  · 检查日期 → 控制概述文字中的检查时间                 │
│  · 落款日期 → 控制文档末尾的签名日期                   │
│                                                      │
│  5S 半月规则：                                        │
│  · 上半月：13-16 号之间，自动避开周六日取第一个工作日   │
│  · 下半月：23-26 号之间，自动避开周六日取第一个工作日   │
│  · 选择后同步更新标题和概述文字中的半月标识             │
└──────────────────────────────────────────────────────┘
```

### 3.3 整改状态与备注
- **上传了整改后照片** → 备注列自动填"已整改"
- **未上传整改后照片** → 备注列留空

### 3.4 照片要求
- 照片压缩到 **< 500KB**
- 表格中图片显示约 **2 英寸宽**（192px @ 96DPI）
- docx.js 中 `ImageRun.transformation` 接收**像素**值，内部自动转 EMU（不能直接传 EMU 值，会被二次放大）
- 一页排版 3-4 行照片，行距固定均匀
- 图片铺满固定单元格

### 3.5 照片上传方式
- **拍照上传**：`<input capture="environment">` 直接调用相机
- **相册选择**：`<input accept="image/*">` 从手机相册选取
- 整改前/整改后两个照片槽位**独立操作**，随时可补拍

### 3.6 AI 润色规则
- **安全类**：末尾追加 `[风险：xxx]`（≤15 个汉字）
- **现场类**（5S/公司）：末尾追加 `[影响：xxx]`（≤15 个汉字）
- 提供 **3 个优化选项**供用户选择
- 选中后可**编辑修改**
- 支持**换一批**重新生成

### 3.7 数据持久化
- 生成报告后**不清除历史草稿**
- 每次操作自动保存到 IndexedDB
- 支持多草稿并存

## 四、完整操作流程

```
┌──────────┐    ┌──────────┐    ┌──────────────┐
│  首页     │───▶│ 条目列表  │───▶│ 新增/编辑条目 │
│ 选类型    │    │ 增删改查  │    │ 拍照+描述    │
│ 续草稿    │    │ 生成报告  │    │ AI润色      │
└──────────┘    └──────────┘    └──────────────┘
                      │                │
                      ▼                ▼
                ┌──────────┐    ┌──────────────┐
                │ 生成预览  │    │  AI润色页     │
                │ 改日期    │    │  3选1+编辑   │
                │ 下载/分享 │    │  换一批      │
                └──────────┘    └──────────────┘
```

### 详细步骤：

1. **首页** — 3 张类型卡片 + 草稿列表，点击选类型或继续草稿
2. **条目列表** — 缩略图 + 描述摘要 + 整改状态，增/删/改/生成按钮
3. **新增条目** — 两个照片槽位 + 问题描述 textarea + 语音按钮 + AI 润色按钮
4. **AI 润色** — 3 个优化选项卡片，选中高亮，可编辑、可换一批
5. **生成预览** — 显示公司/部门/问题数，检查日期 + 落款日期，下载/分享按钮

### 5.5 各模块职责

**app.js** — 全局状态管理
- `state` 对象：`reportType`、`items[]`、`headerInfo{company, department, date, inspectionDate, halfMonth}`、`currentPage`
- 页面路由：`showHome()` → `showItemList()` → `showItemForm()` → `showOptimizePage()` → `showGeneratePage()`
- 事件协调：保存草稿、触发生成、下载分享

**ui.js** — UI 渲染
- `renderHomePage()` — 类型卡片 + 草稿列表
- `renderItemList()` — 条目列表
- `renderItemForm()` — 新增/编辑表单（照片双槽位）
- `renderOptimizePage()` — AI 优化 3 选 1
- `renderGeneratePage()` — 生成预览（日期选择、半月切换）
- `showEditModal()` — 文本编辑弹窗

**docx-gen.js** — Word 生成核心
- `TEMPLATE_CONFIG` — 三种模板的完整排版参数
- `generateDocx()` — 主函数，构建 Document → Packer.toBlob()
- `pickWorkday()` — 范围内取第一个工作日
- `formatDate()` — 中文日期格式（不补零："6月8日" 非 "06月08日"）
- `compressImageForDocx()` — 图片压缩到 <450KB
- `IMG_SIZE_PX = 192` — 图片显示 ~2 英寸宽

**camera-voice.js** — 多媒体输入
- `takePhoto()` — 拍照/相册选择

- `startVoiceRecognition()` — Web Speech API 语音识别

- localStorage：`inspection-presets` 键存公司/部门预设

## 六、关键设计决策与踩坑记录

### 6.1 docx.js ImageRun 尺寸陷阱
- **问题**：传入 EMU 值 1828800（2 英寸）后图片变成 19050 英寸
- **原因**：docx.js `ImageRun.transformation` 接收**像素**，内部自动 `× 9525` 转 EMU。如果传 EMU 值，会被当作像素再次放大 9525 倍
- **解决**：传 `{ width: 192, height: 192 }`（192px @ 96DPI = 2 英寸）

### 6.2 Table gridCol 列宽全为 100
- **问题**：Table 构造函数未传 `columnWidths`
- **解决**：添加 `columnWidths: cfg.columns.map(c => c.width)`

### 6.3 日期格式
- **要求**：中文"2026年6月8日"，不补零（不是"06月08日"）
- **解决**：`formatDate` 直接用数字，不 `padStart`

### 6.4 微信分享
- **问题**：微信内置浏览器不支持 `navigator.share` 分享文件
- **方案**：先触发下载保存到手机，再尝试分享页面链接，提示用户从微信发送文件

### 6.5 微信语音
- **问题**：微信内置浏览器不支持 Web Speech API
- **方案**：检测不支持时明确提示用户用 Safari/Chrome 打开

### 6.6 GitHub Pages 部署
- Vercel 在国内被墙，改用 GitHub Pages
- 单页 HTML 直接部署，无需构建
